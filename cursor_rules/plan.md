# 롤플레잉 일본어 회화 학습 앱 'J-Scenario' 기획서

---

## 1. 프로젝트 개요

### 1-1. 프로젝트명
- **J-Scenario (제이-시나리오)**
- 부제: 살아남아라! 실전 일본어 서바이벌

### 1-2. 프로젝트 목표
- 실제와 같은 돌발 상황 속에서 사용자의 순발력과 회화 능력을 극대화하는 새로운 학습 경험을 제공합니다.
- 게임처럼 즐기는 롤플레잉 방식을 통해 학습 동기를 부여하고, 꾸준한 일본어 말하기 연습을 유도합니다.

### 1-3. 타겟 유저
- **주요 타겟:** 일본어 중급 학습자 (JLPT N3 ~ N2 수준)
- **확장 타겟:** 일본 여행, 유학, 취업을 앞두고 실전 회화 능력을 키우고 싶은 사람

### 1-4. 핵심 컨셉
- **"문자가 아닌, 상황으로 배운다."**
- 사용자에게 예측 불가능한 랜덤 시나리오를 음성, 사진과 함께 제시하고, 사용자는 자신의 목소리로 직접 상황에 대응하며 일본어 회화 실력을 향상시킵니다.

---

## 2. 주요 기능 및 상세 설명

### 2-1. 시나리오 기반 롤플레잉 학습
- **다양한 상황 제공:** 일상, 돌발/위기, 비즈니스, 관계 등 다양한 카테고리의 시나리오를 제공합니다.
- **실감 나는 상황 제시:** AI로 생성된 상황별 이미지와 캐릭터 음성을 통해 높은 몰입감을 제공합니다.
- **명확한 미션 부여:** 각 상황마다 사용자에게 구체적인 미션을 제공하여 학습 목표를 명확히 합니다.

### 2-2. 음성 인식 및 AI 기반 실시간 피드백
- **음성 입력 및 분석:** 사용자의 일본어 발화를 실시간으로 텍스트 변환하고 분석합니다.
- **다각적 평가:** **발음 정확도**, **문법 및 어휘**, **뉘앙스 및 격식(TPO)** 세 가지 기준으로 종합 평가를 내립니다.
- **상세 피드백:** 평가 점수와 함께 구체적인 교정 제안, 더 나은 표현 등의 모범 답안을 제공합니다.

### 2-3. 게임화(Gamification) 요소
- **성장 시스템:** 시나리오 클리어 시 경험치(EXP)를 획득하여 레벨업합니다.
- **도전과제:** 특정 조건을 만족하면 뱃지나 칭호를 부여하여 성취감을 높입니다.

## 3. 사용자 시나리오 (User Flow)

1.  **시나리오 시작**: 사용자가 앱을 실행하여 오늘의 시나리오를 시작합니다.
2.  **상황 제시**: 텍스트, 이미지, 음성으로 돌발 상황이 제시됩니다. (예: "지갑을 잃어버렸다. 경찰에게 상황을 설명해야 한다.")
3.  **미션 확인**: 사용자에게 "분실 경위와 지갑의 특징을 설명하세요." 와 같은 미션이 주어집니다.
4.  **사용자 발화**: 사용자가 마이크 버튼을 누르고 일본어로 말합니다.
5.  **AI 평가 및 반응**: AI가 사용자의 발화를 평가하고, 그에 맞는 반응을 음성으로 들려줍니다.
6.  **결과 확인**: 종합 점수, 상세 피드백, 추천 표현 등을 확인하며 복습합니다.
7.  **보상 획득**: 시나리오 클리어 후 경험치와 재화를 획득합니다.

## 4. 기술 스택 (보유 기술 기반)

### 4-1. 아키텍처
- **Client-Server 모델**: 사용자의 데이터를 서버에 저장하고 AI 분석을 서버에서 처리하는 구조
  - **Client**: Android App
  - **Server**: Backend API Server

### 4-2. Android App (Client)
- **언어**: **Kotlin**
  - Google의 공식 안드로이드 개발 언어로, 안정성과 생산성이 높습니다.
- **UI**: **Jetpack Compose**
  - 선언형 UI 툴킷으로, 더 적은 코드로 직관적이고 아름다운 UI를 빠르게 구축할 수 있습니다.
- **아키텍처**: **MVVM (Model-View-ViewModel)**
  - 데이터와 UI를 분리하여 테스트와 유지보수가 용이한 구조를 채택합니다.
- **주요 라이브러리**:
  - `Retrofit2`: 서버와의 HTTP 통신
  - `Coroutines`: 비동기 처리
  - `Room`: 로컬 데이터베이스 (사용자 설정, 학습 기록 등 캐싱)

### 4-3. Backend (Server)
- **주요 추천 (Python)**:
  - **언어**: **Python**
  - **프레임워크**: **FastAPI**
  - **선택 이유**: 각종 AI/ML 서비스 API(OpenAI, Google Cloud 등)와의 연동이 매우 쉽고, 비동기 처리를 지원하여 빠릅니다. 신속한 프로토타이핑 및 개발에 가장 유리합니다.

### 4-4. AI/ML 서비스 (외부 API 활용)
- **음성 인식 (STT)**: `Google Cloud Speech-to-Text`, `OpenAI Whisper API`
- **음성 합성 (TTS)**: `Google Cloud TTS`, `Naver CLOVA Voice` (다양한 감정 표현)
- **언어 모델 (LLM)**: `OpenAI GPT-4`, `Google Gemini API` (문맥 평가 및 피드백 생성)
- **이미지 생성**: `Stable Diffusion API`, `Midjourney API` (상황 이미지 생성)

### 4-5. Database
- **PostgreSQL** 또는 **MySQL**
  - 정형화된 사용자 데이터, 학습 기록, 시나리오 데이터 등을 관리하기에 적합한 관계형 데이터베이스(RDBMS)를 사용합니다.
  - **참고**: 랭킹 시스템은 구현하지 않음
